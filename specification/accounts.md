# AxleOps Accounting System — Complete Flow Documentation

This document explains how the entire double-entry accounting chain works in AxleOps: what entries are automatic, what entries each role creates manually, how ledgers are maintained, and how data flows from operational events to financial reports and Tally exports.

---

## 1. The Core Principle: Every Rupee Has a Story

AxleOps uses **double-entry bookkeeping**. Every financial event creates a **voucher** (journal entry) where:
- **Debits = Credits** (always balanced)
- Each voucher is **tagged** with Trip ID, Vehicle, Driver, and/or Invoice (linking accounting to operations)
- Each voucher inherits a **branch_id** from its associated trip (enabling branch-level P&L without duplicating ledgers)

### The Accounting Equation
```
Assets = Liabilities + Capital + (Income − Expenses)
```

Every voucher moves money between accounts while keeping this equation balanced.

## 3. Voucher Types

| Voucher Type | Code Prefix | Purpose | Color |
|-------------|-------------|---------|-------|
| **Payment** | `PMT/` | Cash/bank going out to pay expenses, vendors, drivers | 🔴 Red |
| **Receipt** | `RCT/` | Cash/bank coming in from client payments | 🔵 Blue |
| **Sales** | `SLS/` | Invoice raised → revenue recognized, receivable created | 🟢 Green |
| **Purchase** | `PUR/` | Vendor bill recorded → expense booked, payable created | 🟡 Amber |
| **Journal** | `JRN/` | Non-cash entries: depreciation, provisions, adjustments | ℹ️ Info |
| **Contra** | `CNT/` | Cash ↔ Bank transfers (no P&L impact) | ⚪ Grey |
| **Credit Note** | `CRN/` | Reducing client receivable (shortage deduction, returns) | 🟩 Teal |
| **Debit Note** | `DRN/` | Reducing vendor payable (penalty, quality deduction) | 🟩 Teal |

### Voucher Numbering Format
```
[TYPE]/[FY]-[MONTH]/[SERIAL]
Example: SLS/26-03/0042  →  Sales voucher #42 in March of FY 2025-26
```

## 6. How Ledgers Are Maintained

### Opening Balances
When the financial year starts (April 1), opening balances are brought forward from the previous year's closing. The system generates an automatic opening balance voucher.

### Real-Time Balance Updates
Every voucher creation updates the running balance for both the debit and credit ledgers. The Account Ledger page shows this running balance chronologically.

### Monthly Closing Process
1. **Accounts Exec** ensures all pending vouchers are posted (fuel, toll, payments, receipts)
2. **System** runs automatic month-end entries (depreciation, salary accrual)
3. **Finance Controller** reviews Day Book for completeness
4. **Finance Controller** checks Trial Balance (Dr = Cr)
5. **Finance Controller** reviews P&L for margin accuracy
6. **Finance Controller** exports to Tally via Export Center

### Year-End Closing
1. All monthly closes must be complete
2. System calculates net profit and creates closing entry:
   - Dr. Revenue accounts → Cr. P&L Account (close income)
   - Dr. P&L Account → Cr. Expense accounts (close expenses)
   - Net balance transfers to Retained Earnings
3. Opening balances for new year are generated

## 7. Branch-Level Accounting

### The Tagging Approach
Vouchers are **tagged** with `branch_id` (derived from the trip), NOT separated into per-branch ledger copies.

```
Company COA: "Diesel Expense" (one ledger)
├── Voucher PMT/0045: Diesel ₹12,480 → tagged branch: Mumbai
├── Voucher PMT/0048: Diesel ₹9,600  → tagged branch: Pune
├── Voucher PMT/0051: Diesel ₹15,800 → tagged branch: Delhi
```

### Branch P&L
- Generated by filtering vouchers by branch_id tag
- Direct expenses (fuel, toll, hire) are fully attributable to a branch
- Indirect expenses (rent, salary, GPS) can be either:
  - **Untagged** → appear only in consolidated P&L
  - **Allocated** → split proportionally using `(Branch Revenue / Total Revenue) × Indirect Expense`

### What Each Branch Manager Sees
- Account Ledger (filtered to their branch's vouchers)
- Day Book (filtered to their branch)
- Branch P&L and Trip Profitability

### What They Cannot See
- Chart of Accounts (company-wide, not branch-specific)
- Trial Balance (company-wide)
- Export Center (company-wide)
- Balance Sheet, Cash Flow (always consolidated)

### What Gets Exported
1. **Masters** — All 42 ledger accounts with opening balances, group mapping
2. **Sales Vouchers** — Client invoices with line items, GST breakup, HSN code
3. **Receipt Vouchers** — Payment receipts with UTR, TDS, allocation against invoices
4. **Payment Vouchers** — Vendor/driver payments with bank details
5. **Journal Vouchers** — Revenue recognition, fuel/toll, depreciation, adjustments
6. **Credit/Debit Notes** — Shortage deductions, rate corrections with GST reversal

The XML conforms to Tally Prime's import format and has been tested with Tally Prime 4.0.
